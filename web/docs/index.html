Pollen docs

Pollen turns a set of machines into a self-organising mesh that runs WASM workloads, exposes services, and stores blobs. There's no scheduler and no leader. Every node holds the same CRDT and decides locally where workloads belong, where calls go, and what to admit. One static binary, end-to-end mTLS, no shared secrets.

What Pollen is

A peer-to-peer runtime for distributed software. You install pln on a few machines, join them into a cluster, and publish workloads, services, sites, and blobs into it. The cluster places replicas, routes calls to the nearest one, and reorganises around failure. Operators issue capability-bearing certificates rather than push config. Workloads carry their authorisation policy on the spec they're published with.

Pollen is well-suited to:

  • Edge fleets where machines come and go and there's no central control plane.
  • Internal platforms where you want services and workloads addressable by name across NAT.
  • Self-hosters who want to spread load across personal hardware without becoming a sysadmin.
  • Anyone who wants Kubernetes-shaped behaviour without Kubernetes-shaped operations.
