CLI reference

Every pln command grouped by topic. Same content as pln <command> --help, easier to scan. New to Pollen? Run through the Quickstart first; for the model behind the commands, see Concepts.

Contexts

Most pln commands need to know which daemon they're talking to. Named contexts are the primary mechanism. The reserved name default always resolves to the local daemon at $PLN_DIR; add more to point at remote hosts or alternate state directories. See How-to → Multiple clusters.

Aliases. pln ctx is short for pln context, and use is short for switch. Examples below use the short forms; either works.

pln ctx add <name> <target>

Targets containing @ bind to a remote daemon over SSH; absolute paths bind to a local pln dir. Local named contexts get a free UDP port assigned automatically so they coexist with the default-context daemon without colliding on :60611.

--from <dir|default>
For remote contexts: import an existing admin keypair so this shell can act as admin.
pln ctx add prod root@prod.example.com --from default
pln ctx add dev /tmp/pln-dev

pln ctx use <name>

Switch the active context. To use a different context for just one command, set PLN_CONTEXT for that invocation:

pln ctx use prod                        # persistent
PLN_CONTEXT=prod pln status             # one-off

pln ctx ls

List contexts. Active one marked with *.

pln ctx rm <name>

Remove a context. For remote contexts whose identity lives under ~/.pln/contexts/<name>/, the per-context dir is also deleted.

pln ctx current

Print the active context name. Useful in shell prompts.

Per-command override

Three ways to point a single command somewhere other than the active context, without switching it:

$PLN_CONTEXT <name>
Use the named context for this invocation only. Equivalent to pln ctx use scoped to one command.
--dir <path>
State directory. Defaults to ~/.pln, or /var/lib/pln on Linux once pln daemon install has provisioned it. Env PLN_DIR.
-H, --host <user@host>
Tunnel through SSH and run against the remote daemon. Env PLN_HOST.

Lifecycle

pln up

Start the local node. Foreground by default; -d hands off to launchd (macOS) or systemd (Linux). If no credentials exist, auto-initialises a single-node root cluster.

-d, --detach
Run as background service.
--port <n>
Listening UDP port (default 60611).
--ips <ip,…>
Advertise these IPs to peers.
--public
Hint that this node is publicly reachable, making it a relay candidate. Actual reachability is verified by the mesh via NAT probing.
--name <str>
Human-readable node name.
--metrics
Log metrics and trace output at debug level.
pln up
pln up -d --name relay --public

pln down

Stop the background service. State and credentials are preserved.

pln restart

Restart the background service. Use after pln set or pln serve to rebind listeners, or after pln props on root to re-issue the self-signed cert.

pln logs

Tail the background daemon's log (Homebrew log on macOS, journalctl on Linux).

-f, --follow
Stream in real time.
-n, --lines <n>
Number of lines (default 50).

Content

pln seed <source> [name]

Publish content to the mesh. The kind is autodetected from the source: .wasm → workload, directory → static site, anything else → blob (or stdin when source is -). Publishing workloads, static sites, and named blobs requires the publisher capability (or admin, which is a strict superset). Each published resource carries a signed cert anchored to the publisher's delegation cert.

--min-replicas <n>
Workload only. Minimum replicas; default 2. Static sites fan out to every node with static-http enabled; blobs replicate on demand.
--everywhere
Workload only. Place a replica on every node.
--memory, --mem <size>
Workload only. Memory limit (e.g. 64MiB, 1GiB). Default 8MiB; raise it for Go-WASM guests.
--timeout <dur>
Workload only. Per-invocation timeout (e.g. 500ms, 5s).
--allow-prop key=value
Require the caller's delegation-cert property to equal value. Repeatable; all clauses must match. Without any policy flag the resource is open to every authenticated peer. See Concepts → Authorisation.
pln seed ./hello.wasm seeded hello (4f3a7c91) pln seed ./public my-site seeded my-site (12 files, manifest 3a8c7d2e) pln seed ./big.bin payload seeded payload (e892bca7) cat data | pln seed - b1f9a2d4c5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2

pln unseed <name-or-hash>

Stop publishing content. Resolves the arg against current workloads, sites, and blobs; errors if it matches more than one kind.

pln unseed hello
pln unseed ab12cd34       # by hash prefix

pln fetch <name-or-hash>

Force this node to download a blob from a peer that already advertises it. Useful for pre-warming or pinning.

--from <peer-hex>
Force-fetch from a specific peer instead of any holder.
pln fetch payload
pln fetch ab12cd34 --from <peer-hex>

pln call <name-or-hash> <function> [input]

Invoke a WASM function. Routed to the nearest, least-loaded replica via P2C. Input may be the third positional or piped on stdin; output is written raw to stdout.

--timeout <dur>
Overall deadline (default 60s).
--retries <n>
Retry budget for transient overload/unavailability (default 3).
pln call echo echo 'hello world' hello world cat payload.json | pln call hello greet

Networking

pln serve <port> [name]

Register a local TCP (or UDP) port as a mesh service. Requires the publisher capability (or admin) and is persisted to config.yaml. Every register issues a fresh signed cert anchored to the publisher's delegation cert.

--udp
Expose as a UDP service.
--allow-prop key=value
Require the caller's delegation-cert property to equal value. Repeatable; all clauses must match. Without any policy flag the service is open to every authenticated peer. See Concepts → Authorisation.
pln serve 8080 api
pln serve 5353 dns --udp
pln serve 9000 internal --allow-prop team=backend

pln unserve <port|name>

Unregister a local mesh service. Removes it from config.yaml; open tunnels from peers terminate.

pln connect <service> [provider] [local-port]

Open a local listener forwarding to a remote mesh service. Provider disambiguates when multiple peers serve the same name (use the peer-id prefix from pln status). Persisted to config.yaml.

pln connect api                # any provider, default port
pln connect api ab12 9090      # specific peer, local port 9090

pln disconnect <service|local-port> [provider]

Tear down a local tunnel. Identify by service name or local port. Removes from config.yaml.

Cluster

pln init

Initialise a new root cluster on this node. Idempotent if already initialised as root.

--prop key=val
Root node properties; baked into the self-signed cert and persisted to config.yaml. Update later with pln props and restart.
pln init initialized root cluster root_pub: 7c9a4f8d3e2b1c5a6d8e9f0a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c cluster_id: a1b2c3d4e5f6789a1b2c3d4e5f6789a1b2c3d4e5f6789a1b2c3d4e5f6789a1b2 pln init --prop role=primary --prop region=eu

pln join <token>

Enrol this node into the cluster the token was minted for. Starts the daemon by default.

--no-up
Enrol credentials without starting.
--public
Hint that this node is publicly reachable, making it a relay candidate. Actual reachability is verified by the mesh via NAT probing.

pln invite [subject-pub]

Mint a signed invite token. Pass to pln join. Requires admin capability on the issuing node. The default tier is leaf (consume only); pass --publisher or --admin to widen.

--subject <hex>
Bind to a specific node public key. Without it, the token admits any node.
--ttl <dur>
Token validity (default 5m).
--expire-after <dur>
Hard access expiry on the issued cert.
--publisher
Mint the joiner's cert with publisher capability so it can publish workloads, blobs, services, and static sites.
--admin
Mint the joiner's cert with admin capabilities (delegate, admit, publish). The issuer must itself be admin-capable. Mutually exclusive with --publisher.
--prop key=val
Properties baked into the joining peer's cert. Repeatable; accepts key=value, a JSON object, or - to read JSON from stdin.
pln invite --ttl 30m --prop role=worker
pln invite --publisher --subject $(ssh worker pln id)
pln invite --admin --subject $(ssh relay pln id)

pln grant <peer-id>

Issue a fresh delegation cert to a connected peer. Identify by peer-id prefix. Default tier is leaf.

--publisher
Grant publisher capability so the peer can publish resources.
--admin
Delegate admin authority (so the cluster keeps working with root offline). Mutually exclusive with --publisher.
--prop key=val
Cert properties.
pln grant ab12cd34 --prop role=lead --prop team=backend certificate issued (leaf) to ab12cd34 pln grant worker1 --publisher certificate issued (publisher) to 8c4d2f9a pln grant relay1 --admin certificate issued (admin) to ab12cd34

pln deny <peer-id>

Revoke a peer's membership and tear down its connections. Identify by peer-id prefix. Cascades to peers admitted by the denied node.

pln purge

Delete local cluster credentials, CAS, runtime state, and config. Daemon must be down. Prompts unless --yes.

--include-keys
Also wipe the node identity keys.
--yes
Skip confirmation.

pln bootstrap ssh [name=]<target> …

Install Pollen, enrol, and start the daemon on each SSH target in parallel. Linux only. Needs SSH as root or with passwordless sudo. Targets may also be piped from stdin (one per line) using -.

--publisher
Issue publisher capability to every enrolled node.
--admin
Delegate admin authority to every enrolled node. Mutually exclusive with --publisher.
--prop key=val
Properties baked into every enrolled node's cert. Repeatable; accepts key=value, a JSON object, or - to read JSON from stdin.
--relay-port <n>
Relay UDP port to advertise (default 60611).
--expire-after <dur>
Hard access expiry on the issued cert.
--no-up
When the orchestrator also enrols itself, skip starting its local daemon.
pln bootstrap ssh user@host bootstrapping user@host... user@host ok 1 succeeded, 0 failed pln bootstrap ssh relay-eu=root@10.0.0.5 relay-us=root@10.0.0.6 --admin pln bootstrap ssh worker=root@10.0.0.7 --publisher echo "media=alice@10.0.0.5" | pln bootstrap ssh -

Inspection

pln status [nodes|services|seeds|static|blobs]

Render an overview of the cluster from this node's perspective. Optionally restrict to a single section.

--wide
Full peer IDs, plus per-peer metrics (CPU, memory, latency, throughput).
--include-offline
Include offline peers, services, and orphaned blobs.
pln status PEERS NODE STATUS ADDR CPUs CPU MEM TUNNELS LATENCY IN OUT 7c9a4f8d (self) - - 8 2% 18% 1 - - - ab12cd34 online (public) 198.51.100.10:60611 4 1% 8% 1 12ms 0 0 SEEDS NAME HASH STATUS REPLICAS LOCAL UPTIME hello 4f3a7c91 ready 2/2 * 4m

pln id

Print this node's hex-encoded identity public key. Pipe-friendly. Used as the --subject of an invite token.

pln invite --subject $(ssh user@host pln id)

pln version

Show binary version, commit, and build date. --short prints only the version.

Daemon admin

Operations that act on the daemon binary or its background unit, rather than the running node. Operational lifecycle (up/down/restart/logs) stays at the top level.

pln daemon install

Linux only. Creates the pln system user, state dir, drops the systemd unit, enables it. If invoked via sudo, adds the invoking user to the pln group. Idempotent.

pln daemon uninstall

Stop and remove the systemd unit. --purge also wipes /var/lib/pln (irreversible).

pln upgrade (alias pln daemon upgrade)

Upgrade the local pln binary via the platform package manager: Homebrew on macOS, the install script on Linux. Linux preserves the current install method by default: /usr/local/bin/pln stays on tarball installs; /usr/bin/pln uses distro package-manager detection.

--restart
Bounce the background service after upgrading.
--method
Linux install method override: auto or tarball.
pln upgrade --restart
pln upgrade --method tarball

Config

Daemon config lives in config.yaml. Manual edits while the daemon runs will be overwritten. Use pln set for daemon listener addresses, pln props for the local node's identity properties.

pln set [key] [value]

Set a daemon config value. Run with no args to list keys. Omit the value to apply the default. Restart the daemon to rebind listeners.

http
Prometheus metrics listener (default :9090).
static-http
HTTP listener for static sites (default :8080).
control-addr
Control API listener (default :50051); reads $PLN_DIR/control.token.
log-level
Verbosity: debug, info, warn, error.
pln set Available keys: control-addr TCP address for the control API (requires $PLN_DIR/control.token) (default :50051) http HTTP address for Prometheus metrics (default :9090) log-level Log verbosity: debug, info, warn, error (default info) static-http HTTP address for serving static sites (default :8080) pln set http :9100 set http=:9100 (run `pln restart` to apply)

pln unset <key>

Clear a daemon config value, restoring default behaviour.

pln props [key=value …]

List or replace this node's identity properties. Properties are baked into the local cert and surfaced to seeds and policies on every call. Local-only; root nodes only (delegated peers receive properties via pln grant or pln invite --prop).

--clear
Remove all properties. Cannot be combined with positional args.

Replace semantics: positional args define the complete set, so any unlisted key is removed. Changes take effect on the next pln up; root re-issues its self-signed cert from the persisted properties on every start, and the cluster's renewal pipeline keeps it fresh from then on.

pln props role=primary region=eu pln props role=primary region=eu properties updated (run `pln restart` to apply) pln props --clear properties updated (run `pln restart` to apply)

Exit codes

Non-zero codes let scripts react to common failures without parsing error text.

0
success
1
generic error
3
not found (no peer, blob, or service matches the arg)
4
permission denied (socket perms, sudo required)
5
ambiguous (a prefix matched multiple things)
6
unreachable (daemon down, peer offline, network)
