Contexts

Most pln commands need to know which daemon they're talking to. Named contexts are the primary mechanism. The reserved name default always resolves to the local daemon at $PLN_DIR; add more to point at remote hosts or alternate state directories. Aliased as pln ctx.

pln context add <name> <target>

Targets containing @ bind to a remote daemon over SSH; absolute paths bind to a local pln dir. Local named contexts get a free UDP port assigned automatically so they coexist with the default-context daemon without colliding on :60611.

--from <dir|default>
For remote contexts: import an existing admin keypair so this shell can act as admin.
pln context add prod root@prod.example.com --from default
pln context add dev /tmp/pln-dev

pln ctx use <name>

Switch the active context. Aliased as pln context switch. Override per-command via $PLN_CONTEXT.

pln context ls

List contexts. Active one marked with *.

pln context rm <name>

Remove a context. For remote contexts whose identity lives under ~/.pln/contexts/<name>/, the per-context dir is also deleted.

pln context current

Print the active context name. Useful in shell prompts.

Per-command override

Skip the context system for one-offs. Both flags have env equivalents.

--dir <path>
State directory. Defaults to ~/.pln, or /var/lib/pln on Linux once pln daemon install has provisioned it. Env PLN_DIR.
-H, --host <user@host>
Tunnel through SSH and run against the remote daemon. Env PLN_HOST.

Lifecycle

pln up

Start the local node. Foreground by default; -d hands off to launchd (macOS) or systemd (Linux). If no credentials exist, auto-initialises a single-node root cluster.

-d, --detach
Run as background service.
--port <n>
Listening UDP port (default 60611).
--ips <ip,…>
Advertise these IPs to peers.
--public
Hint that this node is publicly reachable, making it a relay candidate. Actual reachability is verified by the mesh via NAT probing.
--name <str>
Human-readable node name.
--metrics
Log metrics and trace output at debug level.
pln up
pln up -d --name relay --public

pln down

Stop the background service. State and credentials are preserved.

pln restart

Restart the background service. Use after pln set or pln serve to rebind listeners.

pln logs

Tail the background daemon's log (Homebrew log on macOS, journalctl on Linux).

-f, --follow
Stream in real time.
-n, --lines <n>
Number of lines (default 50).

Content

pln seed <source> [name]

Publish content to the mesh. Kind is autodetected from the source: .wasm → workload, directory → static site, anything else → blob (or stdin when source is -). Publishing workloads, static sites, and named blobs requires an admin-capable node because each published resource carries a signed cert anchored to the publisher's admin delegation cert.

--min-replicas <n>
Workload only. Minimum replicas; default 2. Static sites fan out to every node with static-http enabled; blobs replicate on demand.
--everywhere
Workload only. Place a replica on every node.
--memory, --mem <size>
Workload only. Memory limit (e.g. 64MiB, 1GiB). Default 64MiB.
--timeout <dur>
Workload only. Per-invocation timeout (e.g. 500ms, 5s).
--allow-caller key=value
Require the caller's delegation-cert attribute to equal value; repeatable. Without any policy flag the resource is open to every authenticated peer.
--allow-caller-in key=v1,v2,…
Require the caller attribute to be one of the listed values; repeatable.
--allow-target name
Workload only. Restrict which exported function names may be invoked; repeatable.
pln seed ./hello.wasm                       # workload
pln seed ./hello.wasm greeter --everywhere
pln seed ./public my-site                   # static site
pln seed ./big.bin payload                  # named blob
cat data | pln seed -                       # anonymous blob

pln unseed <name-or-hash>

Stop publishing content. Resolves the arg against current workloads, sites, and blobs; errors if it matches more than one kind.

pln unseed hello
pln unseed ab12cd34       # by hash prefix

pln fetch <name-or-hash>

Force this node to download a blob from a peer that already advertises it. Useful for pre-warming or pinning.

--from <peer-hex>
Force-fetch from a specific peer instead of any holder.
pln fetch payload
pln fetch ab12cd34 --from <peer-hex>

pln call <name-or-hash> <function> [input]

Invoke a WASM function. Routed to the nearest, least-loaded replica via P2C. Input may be the third positional or piped on stdin; output is written raw to stdout.

--timeout <dur>
Overall deadline (default 60s).
--retries <n>
Retry budget for transient overload/unavailability (default 3).
pln call hello greet '{"name":"world"}'
cat payload.json | pln call hello greet

Networking

pln serve <port> [name]

Register a local TCP (or UDP) port as a mesh service. Requires an admin-capable node and is persisted to config.yaml. Every register issues a fresh signed cert anchored to the publisher's admin delegation cert.

--udp
Expose as a UDP service.
--prop key=val
Publisher property; repeatable. Accepts JSON (--prop '{"public":true}') for non-string values. Stored on the service spec as descriptive metadata.
--allow-caller key=value
Require the caller's delegation-cert attribute to equal value; repeatable. Without any policy flag the service is open to every authenticated peer.
--allow-caller-in key=v1,v2,…
Require the caller attribute to be one of the listed values; repeatable.
pln serve 8080 api
pln serve 5353 dns --udp
pln serve 9000 internal --prop '{"public":true}'

pln unserve <port|name>

Unregister a local mesh service. Removes it from config.yaml; open tunnels from peers terminate.

pln connect <service> [provider] [local-port]

Open a local listener forwarding to a remote mesh service. Provider disambiguates when multiple peers serve the same name (use the peer-id prefix from pln status). Persisted to config.yaml.

pln connect api                # any provider, default port
pln connect api ab12 9090      # specific peer, local port 9090

pln disconnect <service|local-port> [provider]

Tear down a local tunnel. Identify by service name or local port. Removes from config.yaml.

Cluster

pln init

Initialise a new root cluster on this node. Idempotent if already initialised as root.

--prop key=val
Root node properties; baked into the self-signed cert and persisted to config.yaml. Update later by editing config and restarting.
pln init
pln init --prop role=primary --prop region=eu

pln join <token>

Enrol this node into the cluster the token was minted for. Starts the daemon by default.

--no-up
Enrol credentials without starting.
--public
Hint that this node is publicly reachable, making it a relay candidate. Actual reachability is verified by the mesh via NAT probing.

pln invite [subject-pub]

Mint a signed invite token. Pass to pln join.

--subject <hex>
Bind to a specific node public key.
--ttl <dur>
Token validity (default 5m).
--expire-after <dur>
Hard access expiry on the issued cert.
--prop key=val
Cert properties; baked into the joining peer's cert.
pln invite --ttl 30m --prop role=worker
pln invite --subject $(ssh joiner pln id) --prop tier=gold

pln grant <peer-id>

Issue a fresh delegation cert to a connected peer. Identify by peer-id prefix.

--admin
Delegate admin authority (so the cluster keeps working with root offline).
--prop key=val
Cert properties.
pln grant ab12cd34 --prop role=lead --prop team=backend
pln grant relay1 --admin

pln deny <peer-id>

Revoke a peer's membership and tear down its connections. Identify by peer-id prefix.

pln purge

Delete local cluster credentials, CAS, runtime state, and config. Daemon must be down. Prompts unless --yes.

--include-keys
Also wipe the node identity keys.
--yes
Skip confirmation.

pln bootstrap ssh [name=]<target> …

Install Pollen, enrol, and start the daemon on each SSH target in parallel. Linux only. Needs SSH as root or with passwordless sudo.

--admin
Delegate admin authority to the new relay.
--relay-port <n>
Relay UDP port to advertise (default 60611).
--expire-after <dur>
Hard access expiry on the issued cert.
--no-up
When the orchestrator also enrols itself, skip starting its local daemon.
pln bootstrap ssh user@host
pln bootstrap ssh relay-eu=root@10.0.0.5 relay-us=root@10.0.0.6 --admin
echo "media=alice@10.0.0.5" | pln bootstrap ssh -

Inspection

pln status [nodes|services|seeds|static|blobs]

Render an overview of the cluster from this node's perspective. Optionally restrict to a single section.

--wide
Full peer IDs, plus per-peer metrics (CPU, memory, latency, throughput).
--include-offline
Include offline peers, services, and orphaned blobs.
pln status
pln status seeds --wide
pln status nodes --include-offline

pln id

Print this node's hex-encoded identity public key. Pipe-friendly. Used as the --subject of an invite token.

pln invite --subject $(ssh user@host pln id)

pln version

Show binary version, commit, and build date. --short prints only the version.

Daemon admin

Operations that act on the daemon binary or its background unit, rather than the running node. Operational lifecycle (up/down/restart/logs) stays at the top level.

pln daemon install

Linux only. Creates the pln system user, state dir, drops the systemd unit, enables it. If invoked via sudo, adds the invoking user to the pln group. Idempotent.

pln daemon uninstall

Stop and remove the systemd unit. --purge also wipes /var/lib/pln (irreversible).

pln daemon upgrade

Upgrade the local pln binary. Linux preserves the current install method by default: /usr/local/bin/pln stays on tarball installs, while /usr/bin/pln uses distro package-manager detection.

--restart
Bounce the background service after upgrading.
--method
Linux install method override: auto or tarball.

Config

Daemon config lives in config.yaml. Manual edits while the daemon runs will be overwritten. Use pln set.

pln set [key] [value]

Set a daemon config value. Run with no args to list keys. Restart the daemon to rebind listeners.

Available keys: http, static-http, control-addr, log-level.

pln set                     # list keys
pln set http :9100
pln set log-level debug

pln unset <key>

Clear a daemon config value, restoring default behaviour.

Exit codes

Non-zero codes let scripts react to common failures without parsing error text.

0
success
1
generic error
3
not found (no peer, blob, or service matches the arg)
4
permission denied (socket perms, sudo required)
5
ambiguous (a prefix matched multiple things)
6
unreachable (daemon down, peer offline, network)
