Pollen

A self-organising mesh and WASM runtime in a single static binary. Workloads place themselves over a zero-trust mesh; every node holds the same converged state and decides locally.

curl -fsSL https://pln.sh/install.sh | bash

Zero to cluster to loadtest. Errors are nodes applying backpressure.

What's in the box

  • WASM seeds. Deploy with pln seed; artifacts distribute peer-to-peer by hash. A seed invokes another by host call (pln://seed/<name>/<fn>) or by returning a tail_call marker, so authz, routing, and policy can live inside WASM. Authored in Go, Rust, JS, Python, C#, Zig via Extism.
  • Mesh services. pln serve 8080 api here, pln connect api there (or pln://service/<name> from a seed). TCP and UDP, end-to-end mTLS.
  • Static sites & blobs. pln seed ./public publishes a site; pln seed ./file shares a file. Same verb across kinds; autodetected. Content-addressed, gossiped, streamed peer-to-peer over QUIC.
  • Self-organising. No scheduler, no leader, no coordinator. Topology, placement, and routing emerge from local state; calls go to the nearest, least-loaded replica, and replicas migrate toward demand.
  • CRDT-native. A converging document on every node; changes gossip, conflicts resolve.
  • Partition-tolerant. Both sides of a split keep running; state converges on rejoin; survivors rehost workloads from failed nodes.
  • QUIC transport. One multiplexed, encrypted, UDP-based connection per peer carries gossip, services, and seeds. Connections punch direct between peers; otherwise they relay through any cluster node both peers can reach.
  • Cryptographic admission. No shared secrets, no firewall rules. Every link is mTLS; capabilities ride on signed delegation certs.
  • Edge-ready. Pure Go, no CGO. Raspberry Pi to cloud host.
  • Ergonomic. Opinionated defaults, opt-in configuration.

Where it fits

  • Edge fleets where machines come and go and there's no central control plane.
  • Internal platforms where you want services and workloads addressable by name across NAT.
  • Self-hosters spreading load across personal hardware without becoming a sysadmin.
  • Anyone who wants Kubernetes-shaped behaviour without Kubernetes-shaped operations.
